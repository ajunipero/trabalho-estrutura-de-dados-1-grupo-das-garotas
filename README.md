### Abstract
This repository contains the final project developed for the **Data Structures I** course.  
The project consists of a **terminal-based management system implemented in C**, designed to demonstrate the correct use of **singly linked lists**, **dynamic memory allocation**, and **modular programming principles**.

The system models a simplified business scenario involving **clients, products, and purchase operations**, with full support for **CRUD functionalities**.

---

### Objectives
- Apply fundamental concepts of **data structures** using singly linked lists
- Practice **manual memory management** in C
- Develop modular, readable, and maintainable code
- Reinforce parameter passing and abstraction without global variables

---

### System Features
- **Client Management (CRUD)**  
  Identification via CPF and personal data
- **Product Management (CRUD)**  
  Unique product codes, pricing, and stock control
- **Purchase Module**
  - Client-specific shopping cart
  - Item listing with total quantity and total cost

---

### Technical Specifications
- Programming Language: **C**
- Data Structure: **Singly linked lists**
- Memory Management: `malloc`, `calloc`, `realloc`, `free`
- Design Constraints:
  - No global variables
  - List manipulation via function parameters
  - Menu-driven terminal interface
- Version Control: Git repository with incremental group contributions

---

### Execution Environment
- Compiled and executed via terminal
- Code developed and analyzed using **Visual Studio Code**
